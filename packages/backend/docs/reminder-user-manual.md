# リマインド機能 ユーザーマニュアル

## 概要

リマインド機能は、あなたが設定した目標に向けて、毎日タスクをメールでお知らせするサービスです。このマニュアルでは、リマインド機能の使い方を説明します。

## リマインドメールについて

### 配信タイミング

- **配信時刻**: 平日の午前10時（日本時間）
- **配信頻度**: 毎日1回（土日祝日は配信されません）
- **配信条件**: 未完了のタスクがある場合のみ

### メール内容

リマインドメールには以下の情報が含まれます：

1. **目標タイトル**: あなたが設定した目標
2. **今日のタスク**: 選択されたタスク（最大3つ）
3. **所要時間**: 各タスクの推定所要時間
4. **合計時間**: すべてのタスクの合計所要時間
5. **タスクリンク**: タスク詳細ページへのリンク

### メール例

```
こんにちは、山田太郎さん

今日のタスクをお知らせします。

目標: プログラミングスキルの向上

今日のタスク:
• Reactの基礎を学ぶ (30分)
  → タスクを開始する

• TypeScriptの型システムを理解する (45分)
  → タスクを開始する

• 簡単なアプリを作成する (60分)
  → タスクを開始する

合計所要時間: 135分

---
配信停止
```

## タスク選択について

### 気分選択

リマインドメールで送られるタスクは、あなたの「気分選択」に基づいて選ばれます。

#### 「このまま行く」を選択した場合

- **2/3のタスク**: 最近取り組んでいるアクションまたは関連するアクションから選択
- **1/3のタスク**: しばらく取り組んでいない古いタスクから選択

**こんな時におすすめ**:
- 今の調子を維持したい
- 集中して取り組みたいアクションがある
- 流れを変えたくない

#### 「気分を変える」を選択した場合

- **すべてのタスク**: しばらく取り組んでいない古いタスクから選択

**こんな時におすすめ**:
- マンネリを感じている
- 新しいことに挑戦したい
- 停滞しているタスクを進めたい

#### 初回（気分選択なし）

- **すべてのタスク**: ランダムに選択

### タスクの種類

#### 実行タスク

- **特徴**: 一度実行すれば完了するタスク
- **例**: 「プログラムを作成する」「資料を読む」「発表する」
- **リマインド**: 1日最大3つまで

#### 習慣タスク

- **特徴**: 継続的に実行する必要があるタスク
- **例**: 「毎日読書する」「毎日ランニングする」「毎日瞑想する」
- **リマインド**: 週1回必ず含まれる（平日に均等分散）

## タスクの実行

### タスクを開始する

1. リマインドメール内の「タスクを開始する」リンクをクリック
2. タスク詳細ページが開きます
3. タスクの内容を確認して実行します
4. 完了したら「完了」ボタンをクリック

### タスクのステータス

- **未着手**: まだ開始していないタスク
- **進行中**: 現在取り組んでいるタスク
- **完了**: 完了したタスク
- **スキップ**: 今回はスキップしたタスク

### 気分選択の変更

タスク詳細ページで、明日のリマインドに向けて気分選択を変更できます：

1. タスク詳細ページの下部にある「明日の気分選択」セクションを確認
2. 「このまま行く」または「気分を変える」を選択
3. 選択は自動的に保存されます

## リマインド設定

### リマインドを停止する

リマインドメールの配信を停止したい場合：

1. リマインドメールの最下部にある「配信停止」リンクをクリック
2. 確認ページが表示されます
3. 「配信を停止する」ボタンをクリック
4. リマインドの配信が停止されます

**注意**: 配信停止後も、目標やタスクのデータは保持されます。

### リマインドを再開する

配信を停止したリマインドを再開したい場合：

1. 目標管理曼荼羅にログイン
2. 設定ページを開く
3. 「リマインド設定」セクションで「リマインドを有効にする」をクリック
4. 次の平日からリマインドが再開されます

## よくある質問

### Q1. リマインドメールが届きません

**A1**: 以下を確認してください：

1. **迷惑メールフォルダ**: リマインドメールが迷惑メールに分類されていないか確認
2. **メールアドレス**: プロフィール設定で正しいメールアドレスが登録されているか確認
3. **リマインド設定**: リマインドが有効になっているか確認
4. **タスクの有無**: 未完了のタスクがあるか確認

### Q2. タスクが多すぎる/少なすぎる

**A2**: タスクの数は以下のルールで決まります：

- **実行タスク**: 1日最大3つまで
- **習慣タスク**: 週1回必ず含まれる

タスクの数を調整したい場合は、アクションの設定を見直してください。

### Q3. 同じタスクばかりリマインドされる

**A3**: 「気分を変える」を選択してください。しばらく取り組んでいない古いタスクが選ばれます。

### Q4. 週末もリマインドしてほしい

**A4**: 現在、リマインドは平日のみの配信となっています。週末の配信は今後の機能追加で検討しています。

### Q5. リマインド時刻を変更したい

**A5**: 現在、リマインド時刻は午前10時（日本時間）固定となっています。時刻の変更は今後の機能追加で検討しています。

### Q6. メール内のリンクが開けない

**A6**: リンクの有効期限は24時間です。期限切れの場合は、目標管理曼荼羅にログインしてタスクページを開いてください。

### Q7. 配信停止したのにメールが届く

**A7**: 配信停止の反映には最大1営業日かかる場合があります。それ以上経ってもメールが届く場合は、サポートにお問い合わせください。

## トラブルシューティング

### メールが届かない場合

1. **迷惑メールフォルダを確認**
   - `noreply@goal-mandala.com`からのメールを探す
   - 見つかった場合は、「迷惑メールではない」としてマーク

2. **メールアドレスを確認**
   - プロフィール設定で正しいメールアドレスが登録されているか確認
   - 必要に応じて更新

3. **リマインド設定を確認**
   - 設定ページでリマインドが有効になっているか確認
   - 無効の場合は有効化

4. **タスクの有無を確認**
   - 未完了のタスクがあるか確認
   - タスクがない場合はリマインドは送信されません

### リンクが開けない場合

1. **有効期限を確認**
   - リンクの有効期限は24時間です
   - 期限切れの場合は、目標管理曼荼羅にログインしてタスクページを開く

2. **ブラウザを確認**
   - 最新のブラウザを使用しているか確認
   - キャッシュをクリアして再試行

3. **ログイン状態を確認**
   - 目標管理曼荼羅にログインしているか確認
   - ログインしていない場合はログイン

## プライバシーとセキュリティ

### データの取り扱い

- **メールアドレス**: リマインドメールの送信にのみ使用
- **タスク情報**: リマインドメールの生成にのみ使用
- **ログ**: 配信状況の確認と改善のために記録（90日後に削除）

### セキュリティ

- **暗号化**: すべての通信はHTTPSで暗号化
- **トークン**: メール内のリンクは24時間で期限切れ
- **配信停止**: いつでも配信を停止可能

詳細は[プライバシーポリシー](https://goal-mandala.com/privacy)をご確認ください。
