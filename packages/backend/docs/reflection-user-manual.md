# 振り返り機能 ユーザーマニュアル

## 概要

振り返り機能は、目標達成状況を定期的に振り返り、改善点を記録するための機能です。

## 使い方

### 1. 振り返りを作成する

#### 手順

1. マンダラチャート画面で「振り返り」ボタンをクリック
2. 振り返り作成画面が表示されます
3. 以下の項目を入力します：
   - **総括**（必須）: 目標達成状況の総括を記入
   - **惜しかったアクション**（任意）: 進捗80%以上のアクション
   - **思ったより進まなかったアクション**（任意）: 進捗1%〜20%のアクション
   - **未着手となったアクション**（任意）: 進捗0%のアクション
4. 「保存」ボタンをクリック

#### アクション候補の選択

各項目には、進捗率に基づいて自動的に分類されたアクション候補が表示されます。候補から選択することで、簡単に入力できます。

- **惜しかったアクション**: 進捗80%以上のアクション
- **思ったより進まなかったアクション**: 進捗1%〜20%のアクション
- **未着手となったアクション**: 進捗0%のアクション

### 2. 振り返り履歴を確認する

#### 手順

1. マンダラチャート画面で「振り返り履歴」ボタンをクリック
2. 過去の振り返り一覧が表示されます（作成日時降順）
3. 振り返りカードをクリックすると、詳細が表示されます

### 3. 振り返りを編集する

#### 手順

1. 振り返り詳細画面で「編集」ボタンをクリック
2. 振り返り編集画面が表示されます
3. 内容を修正します
4. 「保存」ボタンをクリック

### 4. 振り返りを削除する

#### 手順

1. 振り返り詳細画面で「削除」ボタンをクリック
2. 削除確認ダイアログが表示されます
3. 「削除」ボタンをクリックして確定

**注意**: 削除した振り返りは復元できません。

## FAQ

### Q1. 振り返りはいつ作成すればよいですか？

A1. 振り返りは任意のタイミングで作成できます。以下のタイミングがおすすめです：

- 週次: 毎週末に1週間の振り返り
- 月次: 毎月末に1ヶ月の振り返り
- マイルストーン達成時: 重要な節目での振り返り
- 目標達成時: 目標達成後の総括

### Q2. 振り返りは何回でも作成できますか？

A2. はい、振り返りは何回でも作成できます。過去の振り返りは一覧画面で確認できます。

### Q3. 振り返りを作成しないとどうなりますか？

A3. 振り返りは任意の機能です。作成しなくても目標達成には影響しません。ただし、定期的な振り返りは目標達成の成功率を高めることが知られています。

### Q4. アクション候補が表示されないのはなぜですか？

A4. アクション候補は、進捗率に基づいて自動的に分類されます。以下の場合、候補が表示されません：

- 該当する進捗率のアクションが存在しない
- 全てのアクションが同じ進捗率
- アクションがまだ作成されていない

### Q5. 振り返りを削除すると、目標やアクションも削除されますか？

A5. いいえ、振り返りを削除しても、目標やアクションは削除されません。振り返りのみが削除されます。

### Q6. 振り返りを他のユーザーと共有できますか？

A6. 現在のバージョンでは、振り返りは個人用です。他のユーザーと共有する機能は将来のバージョンで追加予定です。

### Q7. 振り返りをエクスポートできますか？

A7. 現在のバージョンでは、エクスポート機能はありません。将来のバージョンで追加予定です。

### Q8. 振り返りの入力内容に文字数制限はありますか？

A8. はい、以下の文字数制限があります：

- 総括: 5000文字
- 惜しかったアクション: 2000文字
- 思ったより進まなかったアクション: 2000文字
- 未着手となったアクション: 2000文字

### Q9. 振り返りを作成すると、進捗率が変わりますか？

A9. いいえ、振り返りを作成しても進捗率は変わりません。進捗率はタスクの完了状況に基づいて計算されます。

### Q10. 振り返りを作成すると、通知が届きますか？

A10. 現在のバージョンでは、振り返り作成時の通知機能はありません。将来のバージョンで追加予定です。

## トラブルシューティング

### 問題1: 振り返りが保存できない

#### 原因

- 総括が入力されていない（必須項目）
- ネットワークエラー
- サーバーエラー

#### 解決方法

1. 総括が入力されているか確認
2. ネットワーク接続を確認
3. ページをリロードして再試行
4. 問題が解決しない場合は、サポートチームに連絡

- サーバーエラー

## ベストプラクティス

### 1. 定期的な振り返り

目標達成の成功率を高めるために、定期的に振り返りを作成しましょう。

- **週次**: 毎週末に1週間の振り返り
- **月次**: 毎月末に1ヶ月の振り返り

### 2. 具体的な記述

振り返りは具体的に記述しましょう。

- ❌ 悪い例: 「うまくいかなかった」
- ✅ 良い例: 「時間管理がうまくいかず、タスクAを完了できなかった」

### 3. 改善点の明確化

次回に向けた改善点を明確にしましょう。

- ❌ 悪い例: 「もっと頑張る」
- ✅ 良い例: 「毎朝30分早く起きて、タスクAに取り組む」

### 4. アクション候補の活用

アクション候補を活用して、効率的に振り返りを作成しましょう。

### 5. 過去の振り返りの参照

過去の振り返りを参照して、改善の進捗を確認しましょう。
